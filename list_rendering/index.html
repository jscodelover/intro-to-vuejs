List Rendering

There are case when we have to iterate over the data(data can be in array or object or object in array and in various other iterable form). To handle such a case Vuejs provide a v-for directive.

The v-for directive allows us to loop over an iterative data and render data from within it. The syntax within the quotes of the v-for directive may look familiar if you have used JavaScript’s 'for of' or 'for in' before.

<ul>
  <li v-for="point in aboutVue">{{ point }}</li>
</ul>

Since our v-for is inside an <li>, Vue will print out a new <li> for each point in our aboutVue array. If our v-for was inside a <div>, then a <div> would have been printed out for each array item along with its content.

For example:

  1. {{ point }}

const app = new Vue({
  el: "#root",
  data:{
   aboutVue: [
     'Vuejs is javscript framework',
     'Vuejs or js-in-html',
     'Vuejs is reactive in nature' ]
  }
})

EXTRA

1. Inside v-for blocks we have full access to parent scope properties. v-for also supports an optional second argument for the index of the current item.

<ul>
  <li v-for="(item, index) in items">{{ index }} - {{ item }}</li>
</ul>

2. You can also use of as the delimiter instead of in, so that it is closer to JavaScript’s syntax for iterators:

<p v-for="(item, index) of items">{{ index }} - {{ item}}</p>

3. You can also use v-for to iterate through the properties of an object.

<ul>
  <li v-for="(value, name, index) in objVue">{{ index }}. {{ name }}: {{value}}</li>
</ul>

const app = new Vue({
  el: "#root",
  data:{
   objVue: {
     title: 'How to do lists in Vue',
     author: 'jscodelover',
     code: 'github' }
  }
})

Result:

  • {{ index }}. {{ name }}: {{value}}
